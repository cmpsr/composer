Provenance Loan Onboarding API API Reference

Welcome to the Provenance (P8e) Loan Onboarding API Reference.

The LOS API is organized around the REST methodology, and it uses resource-oriented URLs and common HTTP response codes to indicate API errors.

All requests are authenticated using an apikey in the header.

All operations that trigger a P8e contract execution are asynchronous and will return an execution UUID. The status of the contract execution can be monitored using the Contract Execution endpoint.

Example Usage:

  • Onboard a loan using the Onboard Loan Endpoint
  • Save the executionUuid returned by the operation
  • Use the executionUuid to poll the status of the onboarding contract using the Contract Execution Endpoint

The native data format for Provenance is Google Protocol Buffers (protos). The Provenance Domain Model for this API is included. Note that protos use snake_case format, while the JSON representation of the protos uses camelCase.

Please Note: Data examples are all highly truncated and should not be used as complete data models.

Schema Definitions

ContractExecutionState: object

Metadata about the status of a contract execution

contract: string

The name of the smart contract class being executed

contractState: string , x ∈ { INITIATED , REQUESTED , COMPLETED , ADVANCED , ERROR }

The status of the contract execution. INITIATED when contract exec starts, COMPLETED when finished, else ERROR if operation failed.

error: string

If an error occurs, the error message will be supplied

executionUuid: string (uuid)

The identifier for the contract execution instance.

scopeUuid: string (uuid)

The identifier for the test of facts on which the contract is executing. Typically this is a business object identifier, like a loan UUID or document UUID
